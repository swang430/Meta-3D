# MPAC OTA 暗室系统拓扑设计

> **文档版本**: 1.0
> **创建日期**: 2026-01-19
> **状态**: 初稿

## 1. 概述

### 1.1 系统设计目标

本文档定义了 MPAC (Multi-Probe Anechoic Chamber) OTA 测试系统的拓扑结构，支持以下测试场景：

| 测试类型 | 说明 | 信道仿真器模式 |
|---------|------|--------------|
| TRP (Total Radiated Power) | SISO 有效发射功率测量 | 直通 (Bypass) |
| TIS (Total Isotropic Sensitivity) | SISO 有效接收灵敏度测量 | 直通 (Bypass) |
| MIMO OTA | 多天线空口性能测试 | 仿真 (Emulation) |

### 1.2 关键设计决策

| 决策项 | 选择 | 理由 |
|-------|------|------|
| 探头天线 | 无源 + 双工器 | 保证空间互易性，简化校准 |
| 射频前端 | 有源 (LNA + PA) | 支持大型暗室 (R > 3m) 链路预算 |
| 收发分离 | 双工器方案 | 避免收发分离天线的互易性问题 |
| 开关矩阵 | 信道仿真器直通 | 减少系统复杂度，统一信号路径 |

### 1.3 链路预算约束

基于信道仿真器最低输入功率 **-30 dBm** 的要求：

- 无源系统暗室半径上限: **~1.7 m** (不满足车辆测试需求)
- 有源系统 (LNA +20dB): **~17 m** (满足全尺寸车辆测试)

### 4.2 信号流向

**上行 (UL) - DUT 发射方向:**
```
DUT TX ~~~> 探头天线 → 双工器(UL) → LNA → 信道仿真器 UL_In
```

**下行 (DL) - DUT 接收方向:**
```
信道仿真器 DL_Out → PA → 双工器(DL) → 探头天线 ~~~> DUT RX
```

### 4.3 组件参数

| 组件 | 参数 | 典型值 | 说明 |
|-----|------|-------|------|
| **双工器** | 隔离度 | > 25 dB | TX-RX 隔离 |
| | 插入损耗 | < 1 dB | 通带损耗 |
| | 频率范围 | 根据测试频段 | FDD 双工间隔 |
| **LNA** | 增益 | 20 dB | 补偿上行路损 |
| | 噪声系数 | < 2 dB | 保证接收灵敏度 |
| | P1dB | > 0 dBm | 避免过载 |
| **PA** | 增益 | 20 dB | 补偿下行路损 |
| | P1dB | > +20 dBm | 线性工作范围 |
| | OIP3 | > +30 dBm | 多载波线性度 |

## 6. 信号流向定义

### 6.1 完整信号链路

**上行链路 (UL): DUT TX → 基站 RX**
```
┌───────┐      ┌──────┐      ┌───────┐      ┌─────┐      ┌──────────┐      ┌──────┐
│  DUT  │ ~~~> │ 探头  │  →   │ 双工器 │  →   │ LNA │  →   │ 信道仿真器 │  →   │ 基站  │
│  TX   │      │ 天线  │      │  UL   │      │     │      │  UL_In   │      │  RX  │
└───────┘      └──────┘      └───────┘      └─────┘      └──────────┘      └──────┘
   +20dBm        -FSPL         -1dB         +20dB          处理           测量
```

**下行链路 (DL): 基站 TX → DUT RX**
```
┌──────┐      ┌──────────┐      ┌─────┐      ┌───────┐      ┌──────┐      ┌───────┐
│ 基站  │  →   │ 信道仿真器 │  →   │  PA │  →   │ 双工器 │  →   │ 探头  │ ~~~> │  DUT  │
│  TX  │      │  DL_Out  │      │     │      │  DL   │      │ 天线  │      │  RX   │
└──────┘      └──────────┘      └─────┘      └───────┘      └──────┘      └───────┘
  +10dBm          处理          +20dB         -1dB          -FSPL        -100dBm
                                                                        (灵敏度)
```

### 6.2 功率电平预算 (典型值 @ 3.5 GHz, R=5m)

**上行链路:**
| 节点 | 电平 (dBm) | 说明 |
|-----|-----------|------|
| DUT TX | +20 | 车载终端 TRP |
| 探头接收 (含 FSPL) | -37 | FSPL ≈ 57 dB @ 5m |
| 双工器后 | -38 | 插损 1 dB |
| LNA 后 | -18 | 增益 20 dB |
| 信道仿真器输入 | -18 | > -30 dBm ✓ |

| 节点 | 电平 (dBm) | 说明 |

## 8. 校准项目定义

### 8.1 校准项目清单

| 校准项目 | 测量内容 | 参考点 | 数据用途 |
|---------|---------|-------|---------|
| **探头路损校准** | SGH → 探头天线 空间路径损耗 | SGH (静区中心) | TRP/TIS 测量补偿 |
| **上行链路校准** | 探头天线 → LNA → CE UL_In | 探头输入 | 上行功率校准 |
| **下行链路校准** | CE DL_Out → PA → 探头天线 | CE 输出 | 下行功率校准 |
| **信道仿真器直通校准** | CE 内部直通路径 | CE 端口 | 系统增益校准 |

### 8.2 校准数据结构

```python
class ProbeCalibrationData:
    probe_id: int                    # 探头编号 (1-32)
    polarization: str                # "V" or "H"
    frequency_ghz: float             # 校准频率

    # 探头路损 (空间路径)
    path_loss_db: float              # SGH → Probe 路损

    # 上行链路增益 (含 LNA)
    ul_gain_db: float                # Probe → CE UL_In

    # 下行链路增益 (含 PA)
    dl_gain_db: float                # CE DL_Out → Probe

    # 元数据
    calibrated_at: datetime
    valid_until: datetime
    temperature_c: float
```

### 8.3 校准流程 (简化)

```
1. 探头路损校准
   ├─ SGH 放置于静区中心
   ├─ VNA Port1 → SGH
   ├─ 探头 → VNA Port2
   └─ 扫描所有探头，记录 S21

2. 上行链路校准
   ├─ 信号注入探头端 (或使用 SGH)
   ├─ 测量 LNA 输出
   └─ 记录 UL 链路增益

3. 下行链路校准
   ├─ 信号注入 CE DL_Out
   ├─ 测量探头辐射 (SGH 接收)
   └─ 记录 DL 链路增益

4. 系统验证
   ├─ 端到端直通测试
   └─ 比对预期值
```

| **余量** | **+55** | **dB** | 充足 ✓ |

### 9.3 暗室尺寸约束

基于上行链路（限制因素）:

```
CE 最低输入: -30 dBm
DUT TX: +20 dBm
系统增益 (LNA - 损耗): +20 - 5 - 1 + 8 = +22 dB

允许 FSPL = 20 + 22 - (-30) = 72 dB

@ 3.5 GHz:
FSPL = 20·log₁₀(d) + 43.3 = 72
d = 10^((72-43.3)/20) = 10^1.435 ≈ 27 m

实际考虑余量 (10 dB): d ≈ 8.5 m
```

**结论: 有源系统支持暗室半径 5~8 m，满足全尺寸车辆测试需求。**
